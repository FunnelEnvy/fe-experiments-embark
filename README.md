# FunnelEnvy Experiment Code for Embark
## Overview
This repo stores experiment code and is used for delivery and review.

## General Process
1. Code should be delivered via commit to a new branch
1. Code should be in a root directory named using experiment ID
1. Sub-directories should be used consistently per client specifications. Some possible use cases:
   1. /minified for minified code
   1. /assets for delivery of non-code files: media assets, etc.
1. All code for initial development should be submitted as a pull request
1. Assign code reviewers per client instructions
1. The pull request URL should be linked to the Initial Build/Development subtask (You may optionally use the ClickUp→GitHub integration which supports branch and PR creation from the ClickUp task)
1. FunnelEnvy and Client team members (developer, PM) should be receiving notifications on the PR for follow-up either by:
   1. Authoring the PR → this should be the default case I think
   1. Being mentioned in a PR comment → if there is feedback
   1. Subscribing to the PR (button in the side sidebar) → if a 3rd party developer submitted
